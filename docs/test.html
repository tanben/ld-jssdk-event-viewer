LD Event Viewer – Demo

The Event Viewer shows you what the LaunchDarkly JavaScript SDK is doing under the hood — which feature flags it evaluated, what user context it sent, any custom tracking events, and the real-time streaming connection it uses to receive flag updates. This page simulates that SDK traffic so you can try the viewer without setting up a real LD environment.

2. Simulate SDK traffic

Each button fires a mock request that the viewer will intercept and display.

Send All fires every event type below in sequence: a flag evaluation request, an identify call, feature & summary events, custom tracking events, conversion goals, and a streaming connection. Good for seeing the full picture in one click.
Flag evaluations happen when the SDK initializes. It sends the current user context to LaunchDarkly and gets back every feature flag value — booleans, strings, numbers, JSON objects. Each flag includes the variation served and the reason it was chosen: targeting rule match, percentage rollout, individual user target, fallthrough default, etc. The viewer shows all of this in a sortable table.
Identify tells the SDK who the current user is. Your app calls identify() with a context object containing a key, name, email, and any custom attributes. The SDK posts this to LaunchDarkly so targeting rules can evaluate against it. The viewer shows the full context payload so you can verify the right attributes are being sent.
Custom events are arbitrary tracking calls your app makes via track('event-key'). You can attach a numeric metricValue (e.g. page load time in ms) and a JSON data payload. LaunchDarkly uses these to measure experiment results — conversion rates, revenue, performance metrics, etc. The viewer shows the event key, value, and any attached data.
Conversion goals (also called "metric definitions") tell the SDK what user actions to watch for. Each goal has a kind — pageview (matches by URL pattern) or click (matches by CSS selector + URL). The SDK fetches these on init and auto-fires events when a match occurs. The viewer validates each goal against the current page: does the URL match? Does the CSS selector find an element?
Streaming is how the SDK receives flag updates in real time. It opens a Server-Sent Events (SSE) connection to LaunchDarkly's edge servers. When someone changes a flag in the dashboard, a put or patch message arrives here within milliseconds — no polling or page reload needed. The viewer shows when the connection opens, any messages received, and connection errors.
